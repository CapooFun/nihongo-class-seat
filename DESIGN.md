# 語学学校クラス座席表 — 設計文書

## 1. 项目概述

### 1.1 项目名称
**nihongo-class-seat** — 日本语言学校班级座位表 Web App

### 1.2 目标用户
日本语言学校的同班同学（约 20-30 人），通过手机浏览器打开查看。

### 1.3 核心价值
- 同学打开手机即可看到 **今天的日期、星期几、是否休息**
- 一眼看到 **今天的老师是谁**
- 看到教室座位图，知道 **每个位置坐的是谁**
- 点击同学/老师可以弹窗查看 **补充信息**（国籍、兴趣、一句话介绍等）

## 2. 设计风格

### 2.1 视觉基调
延续 capoofun.github.io 的 **深色科技蓝** 风格：

| 变量 | 值 | 用途 |
|------|-----|------|
| `--bg-deep` | `#071019` | 页面背景 |
| `--bg-card` | `rgba(15, 30, 55, 0.85)` | 卡片/面板背景 |
| `--border` | `rgba(160, 191, 255, 0.16)` | 卡片边框 |
| `--text-primary` | `#f4f8ff` | 主文字 |
| `--text-secondary` | `#a7b3c9` | 次要文字 |
| `--accent` | `#7cc7ff` | 强调色（链接、标签） |
| `--accent-warm` | `#ffd666` | 暖色强调（黄字/特殊标记） |
| `--seat-empty` | `rgba(255,255,255,0.05)` | 空座位 |
| `--seat-filled` | `rgba(124, 199, 255, 0.12)` | 有人的座位 |
| `--radius` | `16px` | 卡片圆角 |
| `--radius-sm` | `8px` | 小元素圆角 |

### 2.2 字体
```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI",
             "Hiragino Kaku Gothic ProN", "PingFang SC",
             "Microsoft YaHei", sans-serif;
```
需要良好支持 **日文假名 + 汉字 + 英文**。

### 2.3 动效
- 页面加载：卡片从下方淡入（`fadeInUp`，0.3s ease）
- 弹窗：背景遮罩淡入 + 内容从底部滑入（移动端友好）
- 座位 hover/tap：轻微放大 `scale(1.05)` + 边框高亮
- 平滑滚动：`scroll-behavior: smooth`

## 5. 组件详细设计

### 5.1 Header — 今日信息卡片

**功能：**
- 自动根据 `new Date()` 获取今天日期
- 格式化为日文格式：`2026年4月11日（土）`
- 匹配 `SCHEDULE.days` 找到今天的安排
- 显示上课/休息状态（胶囊标签）
- 显示今日老师名字（可点击）

**视觉：**
- 深色卡片，居中，圆角 16px
- 日期用大字号 `clamp(1.2rem, 4vw, 1.6rem)`
- 状态用 pill 标签：休息=红底，上课=绿底
- 老师名字用 `--accent` 色，带下划线提示可点击

### 5.2 周课程表

**功能：**
- 横向展示周一到周五的课程安排
- 每列显示：星期、日期、老师名、是否休息
- 今天对应的列 **高亮显示**（边框发光 + 背景加亮）
- 休息日列显示灰色 + "休み" 标记
- 手机端横向可滑动（`overflow-x: auto` + `scroll-snap`）

**视觉：**
- 5 列网格卡片
- 当天列：`--accent` 色边框 + 微发光 `box-shadow`
- 休息日：降低不透明度，显示斜线纹理

### 5.3 座位表（核心区域）

**功能：**
- 顶部显示讲台/黑板标识
- 下方为 N×M 网格，每格代表一个座位
- 有人的座位显示：名字（日文假名）+ 可选的颜色标记
- 空座位显示为暗色空格
- 点击有人的座位 → 弹出该同学的信息弹窗

**视觉：**
- 网格使用 CSS Grid：`grid-template-columns: repeat(cols, 1fr)`
- 每个座位格子：正方形（`aspect-ratio: 1`），圆角 8px
- 有人的座位：`--seat-filled` 背景，名字居中
- 名字可以有自定义颜色（如图中的"黄字"）
- 讲台用长条形卡片，居中，带渐变背景

**响应式：**
- 手机竖屏（< 400px）：格子最小 56px，允许横向滚动
- 手机横屏 / 平板：格子自适应填满宽度

**触发：** 点击同学座位 或 点击老师名字

## 6. 技术方案

### 6.1 技术栈
| 层 | 选择 | 理由 |
|----|------|------|
| 结构 | HTML5 | 零依赖，GitHub Pages 直接托管 |
| 样式 | CSS3 (原生变量) | 与 capoofun 风格一致，无需构建 |
| 逻辑 | Vanilla JavaScript (ES6+) | 轻量，手机加载快，无框架开销 |
| 数据 | `data.js` 静态文件 | 无后端，同学自行更新推送即可 |
| 部署 | GitHub Pages | 免费、稳定、HTTPS |

### 6.2 文件结构
```
nihongo-class-seat/
├── index.html          # 主页面
├── css/
│   └── style.css       # 全部样式
├── js/
│   ├── data.js         # 数据（课表、老师、同学、座位）
│   └── app.js          # 逻辑（渲染、弹窗、日期判断）
├── images/             # 可选的头像/图片资源
│   └── (teacher/student photos)
├── DESIGN.md           # 本设计文档
└── README.md           # 项目说明 & 使用方法
```

### 6.3 无后端方案说明
- 所有数据硬编码在 `data.js` 中
- 更新座位/课表时，修改 `data.js` 并推送到 GitHub
- 页面加载时 JS 自动读取当前日期，匹配对应数据渲染
- 未来可扩展：接入 Google Sheets API 或 JSON 文件实现动态更新

## 7. 交互流程

### 7.1 页面加载流程
```
用户打开网页
  ↓
JS 获取当前日期 → new Date()
  ↓
格式化为日文日期显示
  ↓
匹配 SCHEDULE 中今天的数据
  ↓
渲染 Header（日期 + 状态 + 老师）
  ↓
渲染周课程表（高亮今天）
  ↓
渲染座位表网格
  ↓
绑定点击事件
  ↓
页面就绪 ✅
```

### 7.2 点击座位流程
```
用户点击一个座位格子
  ↓
判断该座位是否有人
  ├─ 无人 → 无反应
  └─ 有人 → 查找 STUDENTS[key]
              ↓
           创建弹窗 DOM
              ↓
           填充同学信息
              ↓
           显示弹窗（动画滑入）
              ↓
           等待关闭操作
              ↓
           移除弹窗 DOM
```

## 8. 响应式设计

| 断点 | 布局调整 |
|------|---------|
| < 360px | 座位格子 48px，允许横滚；课程表纵向堆叠 |
| 360-480px | 座位格子自适应（推荐竖屏使用）；课程表横向滚动 |
| 480-768px | 座位表完整展示；课程表完整展示 |
| > 768px | 居中限宽 600px，桌面端也可用 |
